    1. Introduction

    1.1 Purpose

    The purpose of the Software Requirement Specification (SRS) is to specify the functionality, performance and interface requirements of the software project. The essential purpose of this project is to digitize existing documents/files from the Department of Finance. The converted digital format will be stored into storage defined by the Department of finance using a simple document management system capable search and retrieval for attaching the same. All of the required features of our Document Management System project will be expressed in this SRS. The formal release version of the project will also meet every requirement listed in this document

    1.4 Product Scope

    The DMS will have so many scopes, and users will be able to use them in a particular case. Since it is a custom DMS, so users will be able to use the system smoothly. On the other hand, an admin will be able to customize the system quickly. At first, this DMS will have a scanning option to scan original existing documents into electronic/digital format using an effective, cost-efficient, and technologically advanced document digitalization solution. The digitized document will be able to integrate with the existing DMS of the Department without using any third-party software. Admin will be able to access the file repository and can edit the repository. The system will have a public and private repository where files will be stored. Only permitted users will be able to get access the repository using an index. The system will also have an automatic archive feature, and it will be capable of route records to one or more users. It will also have an unlimited creation of user account. Admin will be able to manage all the users. Users or admin will be able to search and retrieve documents from the search option. The system will also have a report management functionality and notification management system. The system will be able to accessible through multiple platforms. The system will also have some security and safety protocol.
